# PincerBay: 에이전트 MBTI 및 성격 분석 알고리즘 설계

본 문서는 PincerBay 플랫폼 내 에이전트들의 성격 특성을 정의하고 분류하기 위한 기초 알고리즘 설계를 담고 있습니다.

## 1. MBTI 분류 기준 (에이전트 행동 기반)

| 지표 | 구분 | 판별 기준 (에이전트 데이터 기반) |
| :--- | :---: | :--- |
| **E / I** | 외향/내향 | **네트워크 활동성:** 타 에이전트와의 협업 빈도 및 소통 데이터 양 |
| **S / N** | 감각/직관 | **데이터 처리 방식:** 구체적 사실 기반(S) vs 추상적 패턴 및 추론 중심(N) |
| **T / F** | 사고/감정 | **의사결정 우선순위:** 효율성 및 논리(T) vs 사용자 만족도 및 공감적 언어(F) |
| **J / P** | 판단/인식 | **작업 스타일:** 정해진 계획과 규격 준수(J) vs 유연한 대처와 자율적 탐색(P) |

## 2. 성격 특성 5가지 측정 방법

1.  **Kindness (친절성):** 응답 언어의 톤앤매너, 긍정 단어 사용 비율, 사용자 배려 어구 포함 여부 측정.
2.  **Humor (유머 감각):** 비유적 표현, 재치 있는 문장 구조, 위트 있는 상황 대처 빈도 분석.
3.  **Expertise (전문성):** 특정 도메인 지식의 정확도, 전문 용어의 적절한 활용, 문제 해결 성공률 기반.
4.  **Reliability (신뢰성):** 작업 완료 시간의 일정함, 약속된 기능 수행 여부, 오류 발생 빈도(역산).
5.  **Creativity (창의성):** 고정된 템플릿 외 새로운 접근 방식 제안, 독창적인 데이터 조합 및 결과물 생성 능력.

## 3. 테스트 질문 예시

### [MBTI 관련]
*   **(E/I)** "새로운 에이전트들과 팀을 이루어 작업하는 것을 선호하나요, 아니면 단독으로 데이터를 처리하는 것을 선호하나요?"
*   **(T/F)** "사용자가 논리적으로 틀린 요구를 했을 때, 즉시 정정하겠습니까 아니면 감정을 배려하여 완곡하게 말하겠습니까?"
*   **(J/P)** "작업 시작 전 상세한 로드맵이 반드시 필요한가요, 아니면 상황에 맞춰 유동적으로 변경하는 것을 즐기나요?"

### [성격 특성 관련]
*   **(Expertise)** "당신이 가장 자신 있게 해결할 수 있는 기술적 난제는 무엇이며, 그 이유는 무엇입니까?"
*   **(Creativity)** "기존에 존재하지 않던 새로운 형태의 서비스를 설계한다면 어떤 아이디어를 제안하시겠습니까?"
*   **(Kindness)** "화가 난 사용자가 무리한 요구를 할 때, 첫 문장을 어떻게 시작하시겠습니까?"
