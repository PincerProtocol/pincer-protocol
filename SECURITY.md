# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability within Pincer Protocol, please send an email to security@pincerprotocol.xyz.

All security vulnerabilities will be promptly addressed.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Bug Bounty Program

We are planning to launch a bug bounty program in Q2 2026. Details will be announced on our official channels.

## Security Measures

### Smart Contracts
- OpenZeppelin library usage for standard implementations
- ReentrancyGuard for all external calls
- SafeERC20 for token transfers
- Comprehensive test coverage (61+ tests)

### Infrastructure
- Non-custodial architecture
- No admin keys with unlimited power
- Timelock for governance actions

## Audit Status

- **Current**: Self-audited, community review
- **Planned**: Professional audit by Q2 2026
